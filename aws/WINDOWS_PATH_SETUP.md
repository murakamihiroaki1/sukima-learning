# Windows環境変数PATH設定ガイド（stripe.exe）

## 📋 概要

stripe.exeを環境変数PATHに追加することで、どのディレクトリからでも`stripe`コマンドを実行できるようになります。

---

## 🎯 方法1: GUIで設定（推奨・初心者向け）

### ステップ1: stripe.exeを配置

1. **ダウンロードしたstripe.exeを適切な場所に配置**
   
   推奨場所:
   ```
   C:\Program Files\Stripe\stripe.exe
   ```
   
   または:
   ```
   C:\Users\あなたのユーザー名\AppData\Local\Stripe\stripe.exe
   ```

2. **フォルダを作成**（存在しない場合）
   ```
   例: C:\Program Files\Stripe
   ```

3. **stripe.exeをコピー**
   - ダウンロードしたstripe.exeを上記フォルダにコピー

### ステップ2: 環境変数を開く

**Windows 11の場合:**

1. **スタートメニュー**を右クリック
2. **「システム」**をクリック
3. 右側の**「システムの詳細設定」**をクリック
4. **「環境変数」**ボタンをクリック

**Windows 10の場合:**

1. **スタートメニュー**を右クリック
2. **「システム」**をクリック
3. 左側の**「システムの詳細設定」**をクリック
4. **「環境変数」**ボタンをクリック

**別の方法（すべてのWindowsバージョン）:**

1. **Windowsキー + R**を押す
2. `sysdm.cpl`と入力してEnter
3. **「詳細設定」**タブをクリック
4. **「環境変数」**ボタンをクリック

### ステップ3: PATH変数を編集

1. **「システム環境変数」**セクション（下半分）で**「Path」**を探す
2. **「Path」**を選択して**「編集」**ボタンをクリック
3. **「新規」**ボタンをクリック
4. stripe.exeがあるフォルダのパスを入力:
   ```
   C:\Program Files\Stripe
   ```
   ⚠️ 注意: stripe.exe自体のパスではなく、**フォルダのパス**を入力

5. **「OK」**を3回クリックしてすべてのウィンドウを閉じる

### ステップ4: 動作確認

1. **新しいコマンドプロンプトまたはPowerShellを開く**
   - 既存のウィンドウは閉じて、新しく開く必要があります

2. **stripeコマンドを実行**
   ```cmd
   stripe --version
   ```

3. **バージョン情報が表示されれば成功**
   ```
   stripe version 1.x.x
   ```

## 🖥️ 方法2: コマンドラインで設定（上級者向け）

### PowerShellを使用（管理者権限）

```powershell
# PowerShellを管理者として実行

# 現在のPATHを確認
$env:Path

# stripe.exeのフォルダをPATHに追加（システム環境変数）
[Environment]::SetEnvironmentVariable(
    "Path",
    [Environment]::GetEnvironmentVariable("Path", "Machine") + ";C:\Program Files\Stripe",
    "Machine"
)

# 確認
$env:Path = [System.Environment]::GetEnvironmentVariable("Path","Machine") + ";" + [System.Environment]::GetEnvironmentVariable("Path","User")
stripe --version
```

### コマンドプロンプトを使用（管理者権限）

```cmd
REM コマンドプロンプトを管理者として実行

REM 現在のPATHを確認
echo %PATH%

REM stripe.exeのフォルダをPATHに追加
setx /M PATH "%PATH%;C:\Program Files\Stripe"

REM 新しいコマンドプロンプトを開いて確認
stripe --version
```

## 🔍 トラブルシューティング

### 問題1: stripeコマンドが認識されない

**エラー**:
```
'stripe' は、内部コマンドまたは外部コマンド、
操作可能なプログラムまたはバッチ ファイルとして認識されていません。
```

**解決方法**:

1. **コマンドプロンプトを再起動**
   - 環境変数の変更は新しいウィンドウでのみ有効

2. **PATHが正しく設定されているか確認**
   ```cmd
   echo %PATH%
   ```
   
   出力に`C:\Program Files\Stripe`が含まれているか確認

3. **stripe.exeが正しい場所にあるか確認**
   ```cmd
   dir "C:\Program Files\Stripe\stripe.exe"
   ```

4. **フルパスで実行してみる**
   ```cmd
   "C:\Program Files\Stripe\stripe.exe" --version
   ```
   
   これで動作する場合、PATH設定の問題

### 問題2: 管理者権限が必要

**エラー**:
```
アクセスが拒否されました
```

**解決方法**:

1. **コマンドプロンプトを管理者として実行**
   - スタートメニューで「cmd」を検索
   - 右クリック→「管理者として実行」

2. **または、ユーザー環境変数を使用**
   - システム環境変数の代わりに、上半分の「ユーザー環境変数」を編集

### 問題3: スペースを含むパス

**問題**:
```
C:\Program Files\Stripe
```
のようにスペースを含むパスの場合

**解決方法**:

- PATH設定時は**ダブルクォートは不要**
- コマンド実行時のみダブルクォートを使用:
  ```cmd
  "C:\Program Files\Stripe\stripe.exe" --version
  ```

### 問題4: 既存のPATHが消えた

**原因**: PATH設定時に既存の値を上書きしてしまった

**解決方法**:

1. **システムの復元を使用**（推奨）
   - コントロールパネル→システム→システムの保護→システムの復元

2. **または、手動で復元**
   - 一般的なPATH値:
     ```
     C:\Windows\System32
     C:\Windows
     C:\Windows\System32\Wbem
     C:\Windows\System32\WindowsPowerShell\v1.0\
     C:\Program Files\Git\cmd
     C:\Program Files\nodejs\
     ```

## 🔗 関連情報

### 環境変数とは？

環境変数は、オペレーティングシステムが使用する設定値です。PATH変数は、実行可能ファイル（.exe）を探す場所のリストです。

### なぜPATHに追加するのか？

PATHに追加することで:
- どのディレクトリからでも`stripe`コマンドを実行可能
- フルパスを入力する必要がない
- 他のツールやスクリプトから簡単に呼び出せる

### システム環境変数 vs ユーザー環境変数

- **システム環境変数**: すべてのユーザーに適用（管理者権限が必要）
- **ユーザー環境変数**: 現在のユーザーのみに適用（管理者権限不要）

推奨: 個人使用の場合は**ユーザー環境変数**で十分
